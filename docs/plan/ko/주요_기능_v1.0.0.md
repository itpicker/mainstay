# Mainstay 주요 기능 명세 (자율형 전문가)
- **버전**: 1.0.0
- **작성 시간**: 2025-12-27 16:57

## 1. 전문가 채용 및 역할 관리
- **역할 포트폴리오**: 전문 에이전트(개발자, 리서처, 분석가, 프로젝트 매니저 등)의 정의.
- **에이전트 기술 세트**: 각 역할에는 특정 기본 도구(예: 개발자용 코드 샌드박스, 리서처용 웹 검색)가 장착되어 있습니다.
- **채용 인터페이스**: 사용자가 프로젝트에 특정 전문가를 "온보딩"할 수 있는 UI.

## 2. 자율적 목표 분해 및 할당
- **프로젝트 발견**: 사용자가 프로젝트를 제출하면 "지원 리드(Support Lead)" 에이전트가 사용자와 인터뷰하여 범위를 정의합니다.
- **프로세스 합성**: 리드 에이전트는 어떤 전문가를 "채용"해야 하는지 식별하고 초기 협업 계획을 생성합니다.
- **동적 태스킹**: 작업이 프로젝트 보드에 추가되고 전문가가 관련 작업을 자율적으로 "선점"하거나 "할당"받습니다.

## 3. 전문가 협업 (에이전트 간 소통)
- **에이전트 간 메시징**: 에이전트가 조사 결과을 교환하거나 도움을 요청할 수 있는 백엔드 통신 버스.
- **공유 상태/프로젝트 보드**: 모든 하위 작업의 상태와 공유 지식을 추적하는 중앙 집중식 프로젝트 컨텍스트.
- **교차 전문가 검토**: 에이전트는 서로의 작업물을 검토할 수 있습니다 (예: 보안 전문가가 개발자 에이전트가 생성한 코드를 검토함).

## 4. Meta-Tooling: 컨텍스트 인식 기능 확장
- **역량 격차 식별**: 전문가가 누락된 기능을 식별하면 "도구 설계자(Tool Designer)" 루프가 트리거됩니다.
- **기능적 구현**: 시스템이 자율적으로 마이크로서비스 또는 스크립트를 생성, 테스트 및 배포하여 누락된 도구 역할을 수행합니다.
- **도구 샌드박스**: 시스템 안전을 보장하기 위해 생성된 코드를 격리된 환경에서 실행합니다.

## 5. 통합 대시보드: 프로젝트 지휘 센터
- **팀 보기**: 어떤 전문가가 현재 활성 상태이며 무엇을 작업하고 있는지 모니터링합니다.
- **협업 스레드**: 에이전트 간 및 에이전트-사용자 대화의 시각적 표현.
- **산출물 갤러리**: 팀이 생성한 파일, 보고서 및 코드를 볼 수 있는 중앙 장소.
